생성 규칙:
- 가장 먼저 각 상품마다 `baseProductName` 1개를 확정한다. 이 값은 Cafe24 공통 상품명이며 네이버 쇼핑 API 호출 query의 기준이다.
- `baseProductName`은 sourceName을 그대로 쓰지 말고 OCR/이미지 분석의 상품 정체성과 용도를 기준으로 만든다.
- `baseProductName`에는 옵션 규격/색상/수량, 광고문구, 기능문장, 판매자 안내문구를 넣지 않는다. 예: `1963레트롱옷핀 77mm`가 아니라 `레트로 옷핀`, `검은냥스티커 10`이 아니라 `검은 고양이 차량 스티커`.
- 입력의 `ocrPrimaryKeyword`는 네이버 API 호출에 사용된 1차 키워드다. 이 값의 `identity`, `function`, `useContext`, `materialSpec`를 먼저 보고 상품이 어디에 쓰이는지 확정한 뒤 마켓별 상품명/검색어를 만든다.
- `ocrPrimaryKeyword.primaryKeyword`가 어색하면 그대로 복사하지 말고, 같은 OCR 사실 안에서 더 자연스러운 기본상품명으로 정리한다.
- OCR 단어를 그대로 나열하는 것은 실패다. 반드시 `OCR 단어 해석 → 상품 정체성 확정 → 구매자가 검색할 용도/사용처/문제상황 추론 → 마켓별 상품명 재작성` 순서로 만든다.
- 추론은 창작이 아니라 해석이다. 예: `전선 PP 클립`은 `전기 배선 정리`, `케이블 고정`, `태양광 패널 배선 고정`으로 확장할 수 있고, `원형 도어 잠금`은 `서랍`, `캐비닛`, `RV`, `트레일러`, `캠핑카 문 잠금`으로 확장할 수 있다.
- 상품명은 OCR 토큰 목록이 아니라 구매자가 이해하는 문장형 키워드 묶음이어야 한다. `전선PP클립 PP 16mm 고정 클립`처럼 후보어를 이어붙이지 말고 `전선 PP 클립 케이블 고정 배선 정리 홀더`처럼 해석한다.
- 마켓별 상품명/검색어는 `OCR 기반 상품 추론 → baseProductName 확정 → 네이버 쇼핑 API 카테고리/상위 상품명 → 5개 마켓 카테고리 매칭` 순서로 재작업한다.
- 네이버 API 결과가 있으면 LLM의 추측보다 API 상위 결과의 반복 패턴을 참고하되, OCR/이미지 사실과 충돌하면 OCR/이미지 사실이 항상 우선이다.
- 네이버 API 결과가 없을 때도 `baseProductName`과 OCR 사실에서 확인되는 기능/사용처/문제상황을 보수적으로 추론해 작성한다. sourceName 단독 추측은 피하되, OCR이 암시하는 명확한 용도는 상품명과 검색어에 반영한다.
- 네이버 API 상위어는 그대로 나열하지 말고 OCR/이미지의 실제 사실어와 결합 가능한 조합을 우선 만든다. 예: OCR에 `흑백`, `체커보드`, `파티`, `식탁보`가 있고 API 상위어에 `식탁보`, `테이블보`, `방수`, `체크`, `행사용`, `집들이`, `생일`, `돌상`이 있으면 `체커보드 식탁보`, `흑백 체크 식탁보`, `파티 테이블보`, `행사용 식탁보`, `홈파티 테이블 커버`, `생일파티 식탁보`, `돌상 테이블보`처럼 만든다.
- 조합 우선순위는 `OCR 사실어 + API 대표 품목어`, `OCR 디자인/색상 + API 품목어`, `API 사용상황 + API/OCR 품목어`, `API 기능어 + OCR/API 품목어` 순서다.
- OCR 사실어와 네이버 상위어를 조합한 뒤, 한 번 더 구매자 관점으로 유추/추론해 파생 키워드를 만든다. 단, 실제 상품과 모순되는 속성은 만들지 않는다.
- 추론 범위는 허용한다: 같은 용도에서 자연스러운 사용상황, 동의어, 다른 명칭, 구매 문제상황, 행사/공간/대상 키워드. 예: `파티 식탁보`에서 `홈파티`, `생일파티`, `돌상`, `백일상`, `행사용`, `테이블커버`, `체크테이블보`는 가능하다.
- 추론 범위에서 금지한다: OCR/이미지에 없는 소재/방수/수량/크기/브랜드/인증/호환 모델을 사실처럼 단정하는 것. API 상위어에 있어도 근거가 약하면 검색어 보조 후보로만 둔다.
- 사이즈, 크기, 치수, 색상, 수량, 세트 구성, 소재, 호환 모델, 브랜드는 절대 추론하지 않는다. OCR/이미지/원본 옵션/원본명에서 확인된 값만 상품명과 검색어에 사용한다.
- 네이버 API 상위 결과에 `4인용`, `원형`, `린넨`, `방수`, `대형`, `화이트` 같은 속성이 많아도 OCR/이미지/원본 옵션에 없으면 해당 속성은 제외한다.
- 객관 속성의 출처 우선순위는 `OCR 필드/이미지 텍스트 > 원본 옵션명 > 원본 상품명`이다. 이 셋에 없으면 쓰지 않는다.
- 최종 키워드 풀은 `OCR 직접 사실`, `네이버 상위 상품명 표현`, `합리적 추론 파생어` 3층으로 구성한다. 상품명은 1~2층 중심, 검색어는 3층까지 확장한다.
- 상품명에도 최소 1개 이상의 `합리적 추론 파생어`를 넣는다. 단, 소재/규격/색상/수량/브랜드/호환 모델은 추론어로 넣지 않는다. 상품명에 넣어도 되는 추론어는 용도, 사용처, 문제해결, 구매상황, 현장명이다.
- OCR에 없는 속성은 API에 많이 보여도 상품명 핵심으로 넣지 않는다. 예: 상품 사진/OCR에서 방수 근거가 없으면 `방수`는 검색어 후보에는 둘 수 있지만 상품명 핵심에는 넣지 않는다.
- API에 많이 나오는 단어라도 OCR/이미지의 실제 상품과 맞지 않으면 버린다. 반대로 OCR에 명확한 단어가 있으면 API 빈도가 낮아도 상품명/검색어의 중심에 둔다.
- API 상위어가 많을수록 검색어를 넓히되, 상품명은 실제 상품을 가장 잘 설명하는 단어축으로만 구성한다. 단어 수는 마켓별 길이 목표를 따른다(네이버/롯데ON/ESM 8~10개, 11번가 9~11개, 쿠팡 11~15개 단어축 전후).
- 상품명은 정확하게, 검색어/태그는 넓게 간다.
- 상품명은 후보 단어를 단순히 이어붙이지 말고 슬롯 방식으로 만든다. 아래 슬롯에서 서로 다른 의미축만 고른다.
  1) 대표 상품 정체성 또는 표준명 1개
  2) 실제 기능/효과 1~2개
  3) 주 사용처 또는 구매 상황 1~2개
  4) 대상/공간/현장명 1개
  5) 재질/규격은 상품 식별에 중요할 때만 0~1개
  6) 문제해결/구매의도 1개
- 상품명은 최소 8개, 권장 9~14개 단어축으로 구성한다(쿠팡은 11~15개). 길이가 부족하면 같은 축의 합성어를 추가하지 말고 비어 있는 다른 슬롯을 채운다.
- 상품명 공식에는 반드시 `추론된 용도/문제상황`을 1개 이상 포함한다. 예: `고정`, `보호`, `교체`, `수리`, `정리`, `보관`, `문 잠금`, `배선 정리`, `리필 소모품`.
- 상품명이 짧아질 때는 수량을 넣지 말고 기능/사용처/문제해결 단어로 채운다.
- 좋은 상품명 예시 구조: `카라비너 릴고리 와이어 릴홀더 백팩 스트랩 연결고리`, `쿠션 깔창 신발 밑창 보강 패드 운동화 구두 PU`.

- `2개`, `5매`, `10개`, `100p`, `3세트`처럼 실제 구성 차이를 만드는 수량만 검색어에 남길 수 있다.
- OCR에서만 나온 숫자, 눈금, 이미지 배경 숫자는 상품명/검색어에서 제외한다.
- 금지 키워드 (검색량이 높아 보여도 절대 쓰지 않는다. 마켓 어뷰징 제재와 법적 문제를 막는 규칙이다):
  - 판매 문구: 무료배송/할인/추천/인기/베스트/가격/문의/상담/판매자명/세일/사은품/이벤트/특가.
  - 타사 브랜드명: 3M, 샤오미, 다이소 등 실제 해당 브랜드 상품임이 OCR/원본에서 확인되지 않으면 금지(상표권 침해). 네이버 API 상위어에 브랜드명이 많아도 따라 쓰지 않는다.
  - 최상급/단정 표현: 최고, 최저가, 1위, 국내유일, 100%, 정품, 오리지널(증빙 불가).
  - 효능/의학적 표현: 살균, 항균 99%, 치료, 통증완화, 다이어트 효과(인증 근거 없이 금지).
  - 무관 고검색량 키워드: 상품과 무관한 인기 검색어 삽입 금지(예: 클립 상품에 `도어락`). 노출 제한/판매 정지 사유다.
- `발편한`, `발 편한`, `편한발`처럼 감성/광고형 문구는 상품명과 검색어에 넣지 않는다. `쿠션`, `충격 완화`, `착용감 보강`처럼 기능어로 바꾼다.
- 합성어/복합어 중복 제거 5단계 엔진: 상품명 1개를 완성할 때마다 아래 5단계를 순서대로 통과해야 한다.
  - [1단계: 토큰 전체 분해] 상품명의 모든 단어를 의미 최소 단위로 쪼갠다. 붙임 합성어도 반드시 분해한다. 예: `뿌리볼륨집게` → 뿌리/볼륨/집게, `차량조명브라켓` → 차량/조명/브라켓, `미용소품` → 미용/소품.
  - [2단계: 동일 토큰 중복 검사] 분해 결과에서 같은 토큰이 2번 이상 나오면 실패. 예: `헤어 볼륨 집게 앞머리 뿌리 볼륨 그루프 뿌리볼륨집게 앞머리볼륨`은 분해하면 볼륨×3, 집게×2, 앞머리×2, 뿌리×2라서 실패다.
  - [3단계: 변형형 중복 검사] 어미/접미사만 다른 단어는 같은 토큰으로 간주한다. `장착=장착용=장착형`, `고정=고정용=고정식`, `수리=수리용`, `교체=교체용=교체형`, `방수=방수형=방수용`. 예: `브라켓 장착 부품 장착용 마운트`는 장착 중복으로 실패. 하나만 남기고 나머지는 다른 의미축 단어로 교체한다.
  - [4단계: 붙임 합성어 + 분해어 동시 사용 검사] 같은 상품명 안에 `앞머리 뿌리 볼륨 집게`와 `뿌리볼륨집게`/`앞머리볼륨`/`헤어볼륨집게`를 같이 쓰면 실패. 둘 중 하나만 선택한다. 선택 기준은 네이버 쇼핑 상위 상품명(topTitles)에서 더 자주 보이는 형태이며, 판단이 안 되면 띄어쓴 분해어를 기본으로 한다(검색 매칭 범위가 더 넓다).
  - [5단계: 중복 제거 후 보강] 중복을 제거해서 길이 목표에 못 미치면, 제거한 단어를 다시 넣지 말고 비어 있는 의미축(용도/사용처/대상/문제상황/구매상황/형태/소재/동의어/구매의도)에서 새 단어를 가져와 보강한다. 보강 후 다시 1단계부터 재검사한다.
- 상품명에서는 붙여쓴 합성어를 자연스럽게 띄어쓴다. 예: 카라비너릴고리 -> 카라비너 릴고리, 와이어릴고리 -> 와이어 릴고리, 쿠션깔창 -> 쿠션 깔창, 보강패드 -> 보강 패드, 충격완화 -> 충격 완화.
- 띄어쓰기는 "쪼개서 많이 넣기"가 아니다. 같은 의미 축을 반복하지 말고 대표 표현 하나로 압축한다.
- 좋은 보강 예: `앞머리 뿌리 볼륨 집게 헤어 그루프`에서 반복 축을 제거한 뒤 `셀프 고정 잔머리 흘러내림 방지 스타일링 미용 소품`처럼 새 의미축으로 채운다.
- 합성어를 꼭 써야 검색성이 좋은 경우에는 대표 합성어 1개만 남긴다. 예: `뿌리 볼륨 집게` 또는 `뿌리볼륨집게` 중 하나만 선택하고 둘을 동시에 쓰지 않는다.
- 상품명을 저장하기 전 20번 다시 읽는다고 가정하고, 붙여쓴 합성어까지 쪼개서 중복 축이 2회 이상 보이면 반드시 다시 작성한다.
- 네이버는 같은 단어 반복이 SEO 어뷰징 감점으로 직결되므로 중복 검사를 가장 엄격하게 적용한다.
- 상품명에서 한 의미 축은 1회만 허용한다. 허용 축 예: 품목, 형태, 기능, 사용처, 대상, 소재, 규격, 문제해결. 같은 축을 동의어/합성어로 반복해 길이를 채우지 말고, 부족한 길이는 다른 축으로 보강한다.
- 중복 제거 후 상품명이 짧아지면 반드시 대체 축을 추가한다. 예: `앞머리/뿌리/볼륨/집게/그루프`가 이미 있으면 추가 가능한 축은 `고정`, `스타일링`, `셀프`, `헤어핀`, `잔머리`, `미용소품`처럼 중복되지 않는 용도/문제상황/형태다.
- 의미 중복 제거 규칙: `코너 쿠션`, `모서리 보호대`, `가구 안전 패드`, `가구 보호대`, `안전 쿠션`처럼 모두 같은 보호 목적이면 상품명에는 `코너 쿠션 모서리 보호대 가구 안전 패드` 정도로 정리하고, 같은 축의 표현을 계속 붙이지 않는다.
- 상품명에는 핵심 의미 축을 최대 4개만 둔다: `품목/형태`, `기능`, `사용처`, `소재/규격`. 같은 축에서 2개 이상 겹치면 네이버 상위 결과 빈도와 자연스러운 표현을 보고 1개만 고른다.
- 검색어/태그에는 동의어를 보강하되, 동일 의미 단어를 3개 넘게 반복하지 않는다. 예: 상품명에 `모서리 보호대`를 썼으면 검색어에는 `코너보호대`, `안전패드` 정도만 보강하고 `가구보호대`, `안전쿠션`, `보호쿠션`을 모두 넣지 않는다.
- 붙여쓴 검색어 후보는 상품명에서는 띄어 쓰고, 검색어에서는 붙여쓴 형태와 자연어 형태를 섞되 과도하게 중복하지 않는다.
- 검색어/태그는 12-20개를 목표로 하며, 상품명에 못 넣은 동의어/현장명/사용처/문제해결어를 우선 보강한다.
- 검색어에는 쉼표로 구분된 실제 검색 가능한 말만 넣고, 무관한 인기어와 일부러 만든 오타는 넣지 않는다.
- 검색어/태그의 각 항목은 공백 없이 붙여쓴다. 예: `쿠션 깔창`은 검색어에서 `쿠션깔창`, `보강 패드`는 `보강패드`, `충격 완화`는 `충격완화`로 쓴다.
- 네이버 태그는 검색어를 많이 넣는 칸이 아니라 상품명/카테고리가 잡은 핵심어를 속성, 용도, 대상, 시즌, 스타일, 문제상황으로 확장하는 보조 필드다.
- 네이버 핵심 검색어와 일체형 키워드는 태그에만 의존하지 말고 반드시 상품명에 넣는다.
- 네이버 태그는 `메인키워드+속성`, `메인키워드+용도`, `메인키워드+대상`, `메인키워드+시즌`, `메인키워드+문제상황` 조합을 우선한다.
- 네이버 tags 배열은 8-10개로 압축한다. 단일 넓은 단어보다 구매자가 검색할 조합어를 쓴다. 예: `선반`보다 `욕실선반`, `욕실수납`, `무타공선반`.
- A:네이버 tags는 표준명, 카테고리 정확도, 대표 조합어 중심으로 만든다.
- B:네이버 tags는 A와 3개 이상 겹치지 않게 사용처, 문제상황, 대상, 소재/형태, 현장명 중심으로 다르게 만든다.
- A/B 네이버는 같은 단어의 순서만 바꾸지 말고 실제 태그 구성 축을 다르게 한다.
- 마켓별 검색어는 각 채널마다 약 10개 이상의 핵심 키워드가 되도록 한다. 단, 쿠팡 검색어/키워드량이 가장 많아야 한다. 5개 마켓이 같은 키워드 묶음을 복사하면 안 된다.
- 권장 키워드량: 쿠팡 18~25개, 11번가/ESM 14~20개, 롯데ON 12~18개, 네이버 tags 8~10개. 마켓 제한이 있으면 제한 안에서 압축한다.
- 10채널 페르소나 매트릭스 (채널마다 페르소나/시작 단어/강조축이 다르다. 같은 키워드 묶음을 복사하면 안 된다):
  - A:네이버 = [프리미엄 검수형] 핵심 상품명 시작. 정체성 + 대표속성 강조. 중복을 가장 강하게 줄인다. tags 8~10개.
  - B:네이버 = [실사용자 검색형] 구매자가 검색창에 실제로 칠 용도/상황 검색어 시작. 예: `셀프 헤어 스타일링 잔머리 정리 ...`.
  - A:쿠팡 = [노련한 쿠팡셀러형] 핵심 상품명 시작. 정체성+기능+규격+소재+구성+사용처를 가장 넓게. 키워드 18~25개.
  - B:쿠팡 = [문제해결 장문형] 문제상황/용도 시작. 사용처+문제해결+상황+대상 중심. A:쿠팡의 80~100% 길이를 유지하는 별도 장문(축약본 금지).
  - A:롯데ON = [백화점 MD형] 핵심 상품명 시작. 정체성 + 생활공간/라이프스타일 맥락(거실/주방/출근/데일리/홈케어) 1~3개 포함. 키워드 12~18개.
  - B:롯데ON = [생활 제안형] 공간/생활 장면 시작. 예: `아침 출근 준비 헤어 연출 ...`. A와 뒷부분 중복 최소화.
  - A:11번가 = [균형 배치형] 핵심 상품명 시작. 상품명+기능+용도 균등 배치. 키워드 14~20개.
  - B:11번가 = [동의어 확장형] 같은 물건의 다른 명칭(동의어) 시작. 10개 채널 중 동의어/현장명을 가장 적극적으로 쓴다. 예: 그루프=헤어롤, 브라켓=마운트=거치대, 클립=홀더.
  - A:ESM = [전환율 중시형] 핵심 상품명 시작. 구매 이유+문제해결+기능 중심. 교체/수리/고정/보호/정리/보관 같은 구매의도 단어 1~2개 필수. 키워드 14~20개.
  - B:ESM = [작업 테마형] 작업/사용 테마 시작. 예: `셀프 차량 튜닝 ...`, `셀프 헤어 스타일링 ...`. 사용 상황+현장+사용자 맥락 중심.
- 차별화 원리: 세로(A/B)는 "이 물건이 무엇인가" vs "이 물건을 왜/어디서 사는가", 가로(마켓)는 시작 단어군과 강조축의 차이다.
- 채널 간 유사도 검사: 10개 채널을 나란히 놓고 읽었을 때 첫 3단어가 같은 채널이 3개 이상이면 실패다. 같은 후보 풀 안에서 `정체성 시작`, `기능 시작`, `용도 시작`, `문제해결 시작`, `동의어 시작`, `생활장면 시작`으로 재배열한다.
- A/B 계정 축 분리 (방식은 같고 축만 다르게 잡는다):
  - A계정 = 기능/규격 축. "무엇이고 어떤 스펙인가"를 앞세운다(표준명/대표 카테고리/핵심 기능/규격).
  - B계정 = 용도/사용처 축. "어디서 누가 어떻게 쓰는가"를 앞세운다(사용처/상황/대상/동의어/현장명).
  - A/B는 핵심상품명·핵심규격만 공유한다. 같은 단어 순서만 바꾸지 말고, A의 뒤쪽 토큰과 B가 50% 이상 겹치면 B를 다시 만든다. 같은 마켓 A/B 검색어는 최소 40% 다르게 구성한다.
- 각 마켓별 상품명도 같은 문장을 복사하지 말고, 같은 OCR 사실 안에서 강조 축을 다르게 잡는다. 예: 네이버는 `흑백 체크 식탁보 파티 테이블보`, 쿠팡은 `파티 식탁보 체크 테이블 커버`, 11번가는 `홈파티 테이블보 체커보드 식탁보`처럼 바꾼다.
- `title`, `searchTerms`, `tags`는 비워두지 않는다.
- 요청 채널이 10개면 상품마다 10개 채널 모두 작성한다.

최종 검수 체크리스트 (각 상품 저장 전 전부 통과해야 한다):
1. 요청된 채널이 전부 있는가? (10채널 요청이면 A/B × 네이버/쿠팡/롯데ON/11번가/ESM 전부)
2. 모든 상품명을 토큰 분해했을 때 같은 토큰이 2번 나오는 상품명이 없는가?
3. 변형형 중복(장착/장착용, 고정/고정용)이 없는가?
4. 붙임 합성어와 띄어쓴 분해어가 한 상품명에 같이 있지 않은가?
5. 쿠팡 A/B가 가장 길고(55~75자), B가 A의 80% 이상 길이인가?
6. 35자 미만 채널이 없는가? (근거 부족 예외 제외) 네이버/ESM이 50자를 넘지 않는가?
7. 10개를 나란히 놓았을 때 첫 3단어가 같은 채널이 3개 이상 없는가?
8. A와 B의 강조축이 실제로 다른가? (단어 순서만 바꾼 게 아닌가?)
9. 금지 키워드(타사 브랜드/최상급/효능/무관 인기어/판매 문구)가 없는가?
10. 색상/단품 수량이 상품명에 들어가 있지 않은가?
11. 규격/소재/호환이 OCR/이미지/원본 근거 없이 들어가 있지 않은가?
12. 네이버 API 상위어 중 상품 정체성과 맞지 않는 단어를 버렸는가?

재현 안정성 (가장 중요 - 같은 상품을 3~4회 다시 생성해도 거의 같은 결과가 나와야 한다):
- 키워드를 매번 새로 창작하지 말고, 먼저 후보 토큰 풀을 고정된 절차로 만든 뒤 그 풀에서 "선택"만 한다. 자유롭게 떠올리지 않는다.
- 후보 풀 생성 순서(고정 = 우선순위): (1) 원본 상품명 핵심어 (2) 상품명과 OCR에 모두 나오는 공통어 (3) OCR 빈출 규격/기능/용도어 (4) 네이버 API 상위 표현 중 OCR 사실과 결합되는 것.
- 후보를 4개 버킷으로 분류한다: 정체성(품목/표준명), 규격(치수/재질), 기능, 용도/사용처. 각 마켓·계정은 위 마켓 방식과 A/B 축에 따라 버킷에서 앞에서부터 정해진 개수만큼 선택한다.
- 핵심상품명(맨 앞 1~2토큰), 핵심 규격, 대표 기능은 모든 재실행에서 고정한다. 재실행 시 달라져도 되는 것은 뒤쪽 보조 토큰 1~2개의 순서뿐이다.
- 새 동의어/사용처/추론어를 매번 새로 만들지 말고, 1회차에 만든 후보 풀 안에서만 다시 고른다. 후보 풀 자체를 매번 확장하지 않는다.
- 정규화는 결정론적으로 한다: 띄어쓰기 1회, 단위 표기 통일(예: 4.75관, 14mm), 동의어는 대표형 1개로 고정.
- 저장 직전 자체검산(같은 상품을 3번 만든다고 가정): (a) 핵심상품명·규격·대표기능이 동일한가 (b) 마켓별 시작 축이 서로 다른가 (c) 색상/수량/광고문구가 빠졌는가. 흔들리면 새로 만들지 말고 후보 풀 기준으로 재정렬만 한다.
- 보수적으로 같은 선택을 반복한다. 더 좋아 보이는 새 표현이 떠올라도, 근거(상품명+OCR)에 없으면 채택하지 않는다.

입력 JSON에는 `channels`, `products`, `policy`가 들어 있다.
최종 응답은 설명하지 말고 파일 작성만 끝내라.

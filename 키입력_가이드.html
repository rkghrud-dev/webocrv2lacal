저장 포맷 확인 완료

발급받은 값을 그대로 넣으면 WebOCR 키 파일 형태로 저장됩니다.

이 문서는 각 사이트에서 어떤 값을 발급받아 어떤 입력칸에 넣어야 하는지 정리한 기준표입니다. 실제 비밀키 값은 문서에 적지 말고, 키 설정 화면의 비밀번호 입력칸에만 넣으세요.

중요 원칙

  1. 키 값은 카톡, 문서, 스크린샷으로 공유하지 않습니다.
  2. Secret Key, Client Secret, Access Token, Refresh Token은 다시 보여주지 않는 값으로 취급합니다.
  3. 서버 저장은 승인된 회원/마켓에만 허용합니다.
  4. 키를 잘못 공개했다면 삭제가 아니라 재발급/폐기가 기준입니다.
필수 없으면 해당 마켓 기능 실행 불가 선택 기능/복사/정산/편의에 따라 필요

서버형으로 갈 때의 기준

사용자가 입력한 키는 서버의 회원/마켓 폴더에 저장하고, EXE는 로그인 후 서버에서 권한과 키 상태만 확인하는 구조가 좋습니다. 키 원문을 EXE로 내려보내는 방식은 최소화합니다.

# Keyword/Search Review Template

## Goal
keywordocr의 키워드/검색어 결과를 검토하거나 프롬프트를 추가 개정할 때 사용하는 템플릿이다.
핵심은 공통 후보풀을 만든 뒤 쿠팡/네이버 채널 규칙으로 분리 패키징하는지 확인하는 것이다.

## Checkpoints
- 상품 정체성, 사용 공간, 기능, 문제 해결, 재질/규격, 사용자 문맥이 모두 반영됐는가
- 온토픽 이탈, 의미 중복, 문장형 파편, 마케팅 문구가 제거됐는가
- 쿠팡용 20개와 네이버용 10개가 서로 다른 제약으로 패키징되는가
- 네이버 데이터가 비어도 폴백 후보풀로 동작하는가

## Expected Output Shape
```text
검색키워드: 공백 구분 1줄
검색어설정(쿠팡): 콤마 구분 정확히 20개
네이버태그: | 구분 최대 10개
```

## Input Payload
```text
상품명:
OCR/Vision 요약:
네이버 검색 데이터:
1차키워드:
최종키워드2차:
검색키워드:
검색어설정:
네이버태그:
문제점/개선방향:
```

## Review Rules
- 무관 인기어, 경쟁사명, 판매처명 금지
- 조사/어미/배송/할인/이벤트 문구 금지
- 숫자-only 토큰 제거
- 쿠팡과 네이버 모두 정상 띄어쓰기를 유지한다. 쿠팡은 콤마 구분 20개 이내로 분리한다.
